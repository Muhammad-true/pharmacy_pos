# Настройка общей базы данных для нескольких касс

## Текущее расположение БД

По умолчанию база данных создается в:
- **Windows**: `%USERPROFILE%\Documents\data\dorukhonai.db`
- Или в директории приложения, если указан относительный путь в `.env`

## Настройка общей БД для нескольких касс

### Вариант 1: Общий сетевой путь (рекомендуется)

1. **Разместите БД на сервере/складе:**
   - Создайте общую папку на сервере (например, `\\SERVER\Shared\PharmacyDB\`)
   - Скопируйте файл `dorukhonai.db` в эту папку
   - Настройте права доступа: все кассы должны иметь права на чтение/запись

2. **Настройте каждую кассу:**
   
   Создайте файл `.env` в корне проекта каждой кассы:
   
   ```env
   # Путь к общей базе данных на сервере
   DATABASE_PATH=\\SERVER\Shared\PharmacyDB\dorukhonai.db
   
   # Или если сервер имеет IP адрес:
   # DATABASE_PATH=\\192.168.1.100\Shared\PharmacyDB\dorukhonai.db
   
   # Имя кассы (уникальное для каждой)
   CASHIER_NAME=Касса 1
   
   # Окружение
   ENVIRONMENT=production
   ```

3. **Важно для SQLite:**
   - SQLite поддерживает работу по сети, но с ограничениями
   - Рекомендуется использовать **WAL режим** (Write-Ahead Logging) для лучшей поддержки параллельного доступа
   - Убедитесь, что все кассы имеют стабильное сетевое соединение

### Вариант 2: Локальная БД + синхронизация (альтернатива)

Если сетевое соединение нестабильно, можно использовать локальные БД с периодической синхронизацией через API.

## Настройка WAL режима для SQLite

Для лучшей поддержки параллельного доступа SQLite автоматически использует WAL режим при работе с Drift.

## Проверка подключения

После настройки проверьте:
1. Все кассы могут открыть файл БД
2. Изменения с одной кассы видны на других
3. Нет конфликтов при одновременной записи

## Важные замечания

⚠️ **Ограничения SQLite при сетевом доступе:**
- SQLite не предназначен для высоконагруженных сетевых приложений
- При большом количестве одновременных записей могут быть задержки
- Рекомендуется максимум 5-10 одновременных подключений

✅ **Рекомендации:**
- Используйте стабильное сетевое соединение
- Размещайте БД на быстром диске (SSD)
- Регулярно делайте резервные копии БД
- Мониторьте размер файла БД (при необходимости делайте VACUUM)

## Резервное копирование

Рекомендуется настроить автоматическое резервное копирование БД:
- Ежедневные копии на отдельный диск
- Хранение копий за последние 7-30 дней

## Пример структуры для продакшена

```
\\SERVER\Shared\PharmacyDB\
├── dorukhonai.db          # Основная БД
├── dorukhonai.db-wal      # WAL файл (автоматически)
├── dorukhonai.db-shm      # Shared memory (автоматически)
└── backups\               # Резервные копии
    ├── 2024-01-15.db
    ├── 2024-01-16.db
    └── ...
```
