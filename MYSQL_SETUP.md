## 📋 Настройка MySQL сервера

### 1. Создание базы данных

Подключитесь к MySQL и создайте базу данных:

```sql
CREATE DATABASE pharmacy_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 2. Создание пользователя (опционально, для безопасности)

```sql
    CREATE USER 'kassa_user'@'%' IDENTIFIED BY 'kassa023637';
GRANT ALL PRIVILEGES ON pharmacy_db.* TO 'kassa_user'@'%';
FLUSH PRIVILEGES;
```

Или используйте существующего пользователя `root` с паролем `023637`.

### 3. Настройка .env файла

Создайте или обновите файл `.env` в корне проекта:

```env
# Тип базы данных (sqlite или mysql)
DATABASE_TYPE=mysql

# Настройки MySQL
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=023637
MYSQL_DATABASE=pharmacy_db

# Настройки приложения
APP_NAME=Аптека Хушдил - POS
APP_VERSION=1.0.0
LOG_LEVEL=info
CASHIER_NAME=Касса 1
ENVIRONMENT=production
```

### 4. Для нескольких касс

Если MySQL сервер находится на другом компьютере:

```env
# Используйте IP адрес сервера вместо localhost
MYSQL_HOST=192.168.1.100  # IP адрес сервера с MySQL
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=023637
MYSQL_DATABASE=pharmacy_db
```

**Важно:** Убедитесь, что:
- MySQL сервер разрешает удаленные подключения
- Порт 3306 открыт в файрволе
- Пользователь имеет права на подключение с удаленных хостов

## 🔄 Миграция данных из SQLite в MySQL

### Вариант 1: Автоматическая миграция (рекомендуется)

При первом запуске приложения с MySQL, Drift автоматически создаст все таблицы.
Затем можно импортировать данные из SQLite.

### Вариант 2: Ручная миграция

1. **Экспорт данных из SQLite:**
   ```bash
   sqlite3 dorukhonai.db .dump > dump.sql
   ```

2. **Преобразование для MySQL:**
   - Замените `INTEGER PRIMARY KEY AUTOINCREMENT` на `INT AUTO_INCREMENT PRIMARY KEY`
   - Замените `TEXT` на `VARCHAR(255)` или `TEXT`
   - Замените `REAL` на `DOUBLE`
   - Удалите SQLite-специфичные команды

3. **Импорт в MySQL:**
   ```bash
   mysql -u root -p023637 pharmacy_db < dump.sql
   ```

## 🔧 Устранение проблем

### Ошибка подключения

**Проблема:** `Can't connect to MySQL server`

**Решение:**
- Проверьте, что MySQL сервер запущен
- Проверьте правильность хоста и порта
- Проверьте права пользователя
- Проверьте файрвол

### Ошибка доступа

**Проблема:** `Access denied for user`

**Решение:**
- Проверьте правильность логина и пароля
- Убедитесь, что пользователь имеет права на базу данных
- Проверьте, разрешены ли удаленные подключения

### Ошибка создания таблиц

**Проблема:** `Table already exists` или синтаксические ошибки

**Решение:**
- Убедитесь, что база данных пустая или удалите существующие таблицы
- Проверьте версию MySQL (рекомендуется 8.0+)
