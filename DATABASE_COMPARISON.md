# MySQL в продакшене (историческое сравнение с SQLite)

> ⚠️ С ноября 2025 приложение работает **только с MySQL**. Материал ниже оставлен как справка по прежней архитектуре с SQLite.

## 📊 Текущая ситуация

**Текущая архитектура:**
- ✅ SQLite с WAL режимом
- ✅ Общая БД на сетевом диске
- ✅ Drift ORM
- ✅ Поддержка до 5-10 одновременных подключений

## 🔄 SQLite vs MySQL: Сравнение

### SQLite (текущее решение)

**✅ Преимущества:**
- ✅ **Простота** - не требует отдельного сервера
- ✅ **Нулевая конфигурация** - работает "из коробки"
- ✅ **Быстрая работа** для локальных операций
- ✅ **Легковесность** - один файл БД
- ✅ **Надежность** - проверенная временем технология
- ✅ **WAL режим** - хорошая поддержка параллельного чтения

**❌ Недостатки:**
- ❌ **Ограниченная параллельность** - максимум 5-10 одновременных записей
- ❌ **Сетевой доступ** - медленнее при работе по сети
- ❌ **Блокировки** - при записи блокирует всю БД
- ❌ **Масштабируемость** - ограничена для большого количества касс

### MySQL (предлагаемое решение)

**✅ Преимущества:**
- ✅ **Отличная параллельность** - сотни одновременных подключений
- ✅ **Сетевой доступ** - оптимизирован для работы по сети
- ✅ **Транзакции** - лучшая поддержка ACID
- ✅ **Масштабируемость** - легко масштабируется
- ✅ **Производительность** - быстрее при большом количестве операций
- ✅ **Резервное копирование** - встроенные инструменты

**❌ Недостатки:**
- ❌ **Требует сервер** - нужен отдельный MySQL сервер
- ❌ **Конфигурация** - требует настройки сервера
- ❌ **Ресурсы** - требует больше памяти и CPU
- ❌ **Сложность** - сложнее в настройке и поддержке

## 🎯 Рекомендация для вашего случая

### Для 2-3 касс: **SQLite (текущее решение)**
- ✅ Достаточно для ваших нужд
- ✅ Проще в настройке и поддержке
- ✅ Меньше ресурсов
- ✅ WAL режим обеспечивает хорошую параллельность

### Для 5+ касс: **MySQL**
- ✅ Лучшая производительность
- ✅ Больше одновременных подключений
- ✅ Лучшая масштабируемость

## 🔧 Техническая возможность перехода на MySQL

### ✅ Да, переход возможен!

**Drift поддерживает MySQL через:**
- `drift/native` - для нативного доступа
- `mysql1` пакет - драйвер MySQL для Dart

### Что нужно будет изменить:

1. **Зависимости** (`pubspec.yaml`):
   ```yaml
   dependencies:
     drift: ^2.14.1
     mysql1: ^0.20.0  # MySQL драйвер
   ```

2. **Создание соединения** (`database.dart`):
   ```dart
   // Вместо NativeDatabase
   import 'package:mysql1/mysql1.dart';
   import 'package:drift/native.dart';
   
   Future<QueryExecutor> createDatabaseConnection() async {
     final settings = ConnectionSettings(
       host: 'localhost',  // или IP сервера
       port: 3306,
       user: 'pharmacy_user',
       password: 'password',
       db: 'pharmacy_db',
     );
     
     return MySqlConnection(settings);
   }
   ```

3. **Конфигурация** (`.env`):
   ```env
   DATABASE_TYPE=mysql
   MYSQL_HOST=localhost
   MYSQL_PORT=3306
   MYSQL_USER=pharmacy_user
   MYSQL_PASSWORD=password
   MYSQL_DATABASE=pharmacy_db
   ```

4. **Миграции** - останутся те же (Drift поддерживает)

5. **DAO и репозитории** - **НЕ НУЖНО МЕНЯТЬ!** ✅
   - Drift абстрагирует работу с БД
   - Весь код останется прежним

## 📋 План перехода (если решите)

### Этап 1: Подготовка
1. Установить MySQL сервер на сервере/складе
2. Создать базу данных и пользователя
3. Настроить права доступа

### Этап 2: Миграция данных
1. Экспортировать данные из SQLite
2. Импортировать в MySQL
3. Проверить целостность данных

### Этап 3: Обновление кода
1. Добавить поддержку MySQL в `createDatabaseConnection()`
2. Обновить конфигурацию
3. Протестировать на тестовой среде

### Этап 4: Развертывание
1. Обновить все кассы
2. Переключить на MySQL
3. Мониторинг работы

## 💡 Мой совет

**Для вашего случая (2 кассы):**

1. **Оставайтесь на SQLite** - это оптимальное решение:
   - Проще в поддержке
   - Достаточно для 2-3 касс
   - Меньше проблем с настройкой
   - WAL режим обеспечивает хорошую параллельность

2. **Если планируете расширение (5+ касс):**
   - Тогда имеет смысл перейти на MySQL
   - Но это можно сделать позже, когда понадобится

3. **Гибридный подход:**
   - Можно сделать поддержку обоих типов БД
   - Выбор через конфигурацию
   - Переход без изменения кода

## 🚀 Вывод

**Текущее решение (SQLite) оптимально для:**
- ✅ 2-3 кассы
- ✅ Локальная сеть
- ✅ Простота настройки

**MySQL имеет смысл для:**
- ✅ 5+ касс
- ✅ Большая нагрузка
- ✅ Планы на масштабирование

**Переход технически возможен и не сложен**, но для вашего случая **SQLite будет лучше**.
