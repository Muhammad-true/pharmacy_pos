# Руководство по установке двух касс «Дорухона Ман»

Документ рассчитан на обычного пользователя Windows. Выполните шаги последовательно, чтобы настроить одну кассу как сервер (`Касса 1`) и вторую как клиент (`Касса 2`), работающий с той же базой MySQL.

---

## 1. Что находится в общей папке

```
DorukhonaMan/
 ├─ Release/                     ← готовое приложение (Windows)
 │   ├─ dorukhonai_man.exe
 │   ├─ data/
 │   ├─ flutter_assets/
 │   └─ icudtl.dat
 ├─ mysql-installer.exe          ← установщик MySQL Server 8.0 (или архив)
 ├─ my.ini.example               ← пример настроек MySQL
 ├─ create_mysql_tables.sql      ← структура базы
 ├─ insert_test_user.sql         ← тестовый администратор (опционально)
 ├─ .env.kassa1.example          ← настройки для кассы-сервера
 └─ .env.kassa2.example          ← настройки для второй кассы
```

Скопируйте всю папку `DorukhonaMan` на обе машины без изменений.

---

## 2. Подготовка оборудования

1. **Две машины на Windows 10+** (лучше одинаковые).
2. **Общий LAN/роутер** – обе кассы должны видеть друг друга.
3. **Права администратора** – нужны для установки MySQL и настройки файрвола.

---

## 3. Настройка Кассы 1 (сервер + приложение)

### 3.1 Установка MySQL

1. Откройте `mysql-installer.exe`.
2. В установщике выберите «Server only» (Workbench/Extras не нужны).
3. Задайте пароль пользователя `root` или придумайте свой (запишите).
4. В разделе настройки сети оставьте порт `3306` и тип `Standalone MySQL Server`.
5. После завершения убеждаемся, что служба `MySQL80` запущена (Services.msc).

> **Совет:** если используете ZIP-архив, распакуйте в `C:\mysql`, выполните `mysqld --install`, затем `mysqld --initialize-insecure`, и запустите службу `net start mysql`.

### 3.2 Разрешаем удалённые подключения

1. Откройте файл `C:\ProgramData\MySQL\MySQL Server 8.0\my.ini`.
2. Найдите строку `bind-address` – установите `0.0.0.0` и сохраните.
3. Перезапустите службу MySQL: `Win + R` → `services.msc` → `MySQL80` → Restart.
4. Откройте порт:
   - «Панель управления → Брандмауэр Защитника Windows → Дополнительные параметры».
   - Создайте входящее правило TCP, порт 3306, разрешить подключение.

### 3.3 Создание базы и пользователя

1. Откройте «Командную строку» (`Win + R` → `cmd`).
2. Перейдите в папку `DorukhonaMan`: `cd /d D:\...\DorukhonaMan`.
3. Выполните:
   ```
   mysql -u root -p
   ```
4. После ввода пароля выполните команды:
   ```sql
   CREATE DATABASE pharmacy_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   CREATE USER 'kassa_user'@'%' IDENTIFIED BY 'kassa023637';
   GRANT ALL PRIVILEGES ON pharmacy_db.* TO 'kassa_user'@'%';
   FLUSH PRIVILEGES;
   ```
5. Загрузите структуру:
   ```
   mysql -u kassa_user -pkassa023637 pharmacy_db < create_mysql_tables.sql
   ```
6. (Опционально) создайте тестового администратора:
   ```
   mysql -u kassa_user -pkassa023637 pharmacy_db < insert_test_user.sql
   ```

### 3.4 Определяем IP адрес Кассы 1

1. Нажмите `Win + R`, введите `cmd`, нажмите Enter.
2. В окне введите `ipconfig`.
3. Найдите раздел «Ethernet adapter» или «Wireless LAN adapter», строку `IPv4 Address`.
4. Например: `IPv4 Address. . . . . . : 192.168.1.50` – этот IP потребуется Кассе 2.

> **Альтернатива:** «Параметры → Сеть и Интернет → Свойства сети» – там тоже указан IPv4.

### 3.5 Настройка .env для Кассы 1

1. Скопируйте `Release\.env.kassa1.example` в `Release\.env`.
2. Заполните:
   ```
   DATABASE_TYPE=mysql
   MYSQL_HOST=localhost
   MYSQL_PORT=3306
   MYSQL_USER=kassa_user
   MYSQL_PASSWORD=kassa023637
   MYSQL_DATABASE=pharmacy_db
   CASHIER_NAME=Касса 1
   ENVIRONMENT=production
   ```
3. Сохраните файл в том же каталоге, где лежит `dorukhonai_man.exe`.

### 3.6 Запуск приложения на Кассе 1

1. Перейдите в `Release/`.
2. Дважды нажмите `dorukhonai_man.exe`.
3. При первом запуске приложение подключится к уже созданной базе – таблицы не будут создаваться автоматически, всё берётся из импортированной схемы.

---

## 4. Настройка Кассы 2 (клиент)

### 4.1 Подготовка

1. Скопируйте ту же папку `DorukhonaMan` на Кассу 2 (можно по USB или через сеть).
2. В разделе `Release/` убедитесь, что есть `dorukhonai_man.exe` и `.env.kassa2.example`.

### 4.2 Настройка .env

1. Скопируйте `.env.kassa2.example` в `.env`.
2. Укажите IP адрес Кассы 1:
   ```
   DATABASE_TYPE=mysql
   MYSQL_HOST=192.168.1.50      # замените на фактический IPv4 Кассы 1
   MYSQL_PORT=3306
   MYSQL_USER=kassa_user
   MYSQL_PASSWORD=kassa023637
   MYSQL_DATABASE=pharmacy_db
   CASHIER_NAME=Касса 2
   ENVIRONMENT=production
   ```
3. Сохраните файл.

### 4.3 Проверка связи

1. На Кассе 2 откройте `cmd`.
2. Введите `ping 192.168.1.50` (подставьте IP сервера). Если ответы приходят – сеть настроена.
3. Если пинга нет: проверьте кабель/Wi-Fi, разрешения брандмауэра и что Касса 1 включена.

### 4.4 Запуск приложения

1. В `Release/` запустите `dorukhonai_man.exe`.
2. На авторизации используйте те же учётные данные, что созданы в базе (например, администратор из `insert_test_user.sql`).
3. Протестируйте пробитие тестового чека – запись должна появиться и на Кассе 1 (общая БД).

---

## 5. Ежедневные действия

1. **Запуск:** сначала включайте Кассу 1 (MySQL сервер), затем Кассу 2.
2. **Резервные копии:** раз в день (или чаще) запускайте
   ```
   mysqldump -u kassa_user -pkassa023637 pharmacy_db > backup_YYYYMMDD.sql
   ```
   храните на внешнем носителе.
3. **Контроль соединения:** если Касса 2 не подключается – проверьте, запущена ли служба `MySQL80` и открыт ли порт 3306.
4. **Обновления:** при новой версии приложения достаточно заменить содержимое папки `Release/`, не трогая MySQL.

---

## 6. Подсказки и диагностика

| Проблема | Что проверить |
|----------|---------------|
| `Can't connect to MySQL server` | IP, порт 3306, служба MySQL, правило файрвола |
| `Access denied for user` | Логин/пароль в `.env`, права пользователя в MySQL |
| `Host '...' is not allowed` | Пользователь создан с `@'%'`, `bind-address=0.0.0.0` |
| Нет связи по `ping` | Кабель/роутер, одинаковая подсеть (192.168.1.*) |
| Приложение не видит `.env` | Файл `.env` находится не рядом с `dorukhonai_man.exe` |

---

## 7. Как устанавливать обновления и не потерять данные

1. **Подготовка:**
   - Договоритесь с кассирами о времени обновления и попросите закрыть приложение на обеих кассах.
   - На Кассе 1 выполните резервное копирование базы:
     ```
     mysqldump -u kassa_user -pkassa023637 pharmacy_db > backup_YYYYMMDD.sql
     ```
   - Скопируйте текущую папку `Release/` в безопасное место (например, `Release_backup/`) на случай отката.
2. **Установка новой версии:**
   - Получите новую сборку `Release/` (обычно приходят архивом или в общей папке).
   - На Кассе 1 и Кассе 2 удалите файлы внутри старого `Release/`, но **не удаляйте** `.env`.
   - Распакуйте новую сборку в ту же папку. Если приложен новый `.env.example`, используйте его только как подсказку; ваш рабочий `.env` оставьте прежним.
3. **Если изменилась структура БД:**
   - В комплекте обновления будет SQL-скрипт миграции (например, `alter_2025_01.sql`).
   - Запустите его на Кассе 1:
     ```
     mysql -u kassa_user -pkassa023637 pharmacy_db < alter_2025_01.sql
     ```
   - После выполнения убедитесь, что ошибок нет, и сделайте дополнительный бэкап.
4. **Проверка после обновления:**
   - Запустите `dorukhonai_man.exe` на Кассе 1, авторизуйтесь и проверьте историю продаж.
   - Запустите на Кассе 2, сделайте тестовый чек и убедитесь, что запись появилась в базе.
5. **Откат (если что-то пошло не так):**
   - Остановите приложение.
   - Восстановите папку `Release/` из `Release_backup/`.
   - Восстановите базу командой:
     ```
     mysql -u kassa_user -pkassa023637 pharmacy_db < backup_YYYYMMDD.sql
     ```
   - После проверки повторите обновление с учётом исправлений.

При таком подходе данные всегда остаются в MySQL, а клиенты получают новые версии приложения без риска потерь.

---

Теперь любой пользователь может развернуть обе кассы: установить MySQL только на Кассе 1, указать её IP на Кассе 2 и сразу работать с общей базой без дополнительных инструментов.
